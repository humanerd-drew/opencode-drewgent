# Drewgent 작업 로그 - 2026-04-15

## Cloudflare Tunnel 설정 (실패)

### 문제
- Cloudflare Tunnel을 `humanerd` 계정이 아닌 다른 계정(seo-i.com)에 생성됨
- `agent.humanerd.kr` 도메인이 Tunnel과 연결되지 않음
- Mac DNS가 Tailscale에 가로채짐 (IPv6 주소 반환)

### 원인
- `cloudflared tunnel create` CLI가 계정을 구분하지 않고 기본 계정에 Tunnel 생성
- API Token에 Tunnel 관리 권한 없음

### 시도한 방법
1. API Token으로 Tunnel 목록 조회 → 권한 오류 (Authentication error)
2. 여러 계정에서 Tunnel 생성 시도 → 모두 잘못된 계정에 생성됨
3. `cloudflared tunnel route dns` CLI → 잘못된 도메인으로 생성 시도
4. Mac `/etc/hosts`에 직접 IP 등록 → Tailscale DNS 우회 불가

### 교훈
- Cloudflare Tunnel은 **Dashboard에서 직접 생성**하고 **올바른 계정에 속하는지 확인** 필수
- API Token에 **Account-level Tunnel 관리 권한** 필요
- `config_src: local` vs `cloudflare` 차이 이해 필요
