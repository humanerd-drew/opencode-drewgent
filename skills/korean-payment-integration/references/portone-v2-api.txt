# PortOne V2 SDK requestPayment — Full Parameter Reference

Source: @portone/browser-sdk v0.1.7 type definitions (PaymentRequestBase)

## Required Fields
| Field | Type | Description |
|-------|------|-------------|
| storeId | string | 상점 ID (admin.portone.io → 연동정보) |
| paymentId | string | 고유 주문번호 (중복 시 실패) |
| orderName | string | 주문명 |
| totalAmount | number | 결제 금액 (KRW=1배, USD=100배, JPY=1배) |
| currency | 'KRW' \| 'USD' \| ... | ISO 4217 통화코드 |
| payMethod | 'CARD' \| 'EASY_PAY' \| 'TRANSFER' \| ... | 결제수단 |

## Optional Fields
| Field | Type | Description |
|-------|------|-------------|

| forceRedirect | boolean | 강제 리디렉션 |
| noticeUrls | string[] | 웹훅 수신 URL |
| appScheme | string | WebView 앱 복귀 URL scheme |
| locale | Locale | UI 언어 |
| customData | object | 커스텀 JSON |
| productType | ProductType | 상품 유형 |
| offerPeriod | OfferPeriod | 서비스 제공 기간 |
| bypass | PaymentBypass | PG사 bypass 값 |
| easyPay.easyPayProvider | 'NAVERPAY' \| 'KAKAOPAY' \| ... | 간편결제 직접 호출 |

## PortOne REST API (Server-side)

### Payment Inquiry
```
GET https://api.portone.io/payments/{paymentId}
Authorization: PortOne {API_SECRET}
```
Response status: 'PAID' | 'READY' | 'FAILED' | 'CANCELLED'

### Payment Confirm (manual approval only)
```
POST https://api.portone.io/payments/{paymentId}/confirm
Authorization: PortOne {API_SECRET}
```
