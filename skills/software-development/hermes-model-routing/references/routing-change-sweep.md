# Post-Routing-Change Cleanup Sweep

When model routing changes (new provider, new default model), old model names
linger in configs, skills, AGENTS.md, and reference docs. A systematic sweep
is required — partial fixes leave misleading references that future agents
will interpret as current routing.

## Procedure

### 1. Define the old patterns

Collect all variations of the old model name(s) that appeared in the system:

```
minimax/m2.7, minimax-m2.7, MiniMax-M2.7, MiniMax-M2.5
minimax/m3, minimax-m3, MiniMax-M3, MiniMax M3
anthropic/claude-opus-4.6, claude-opus-4.6
anthropic/claude-sonnet-4, claude-sonnet-4.6
```

Include regex for common case/separator variations.

### 2. Sweep all active files (not just config.yaml)

Checklist of file categories (grep each):

- **Config files**: `~/.hermes/config.yaml`, `~/.drewgent/config.yaml`,
  `~/.drewgent/P5-ego/config/config.yaml`
- **Agent guidance**: `~/.drewgent/AGENTS.md`, `~/.drewgent/skills/*/SKILL.md`
- **Reference docs**: under `skills/*/references/`
- **Skill frontmatter and body**: any skill that mentions models
- **Cron jobs**: `~/.drewgent/cron/jobs.json` — check `model` field

Do NOT sweep (these are not active guidance):

- **Session archives**: `P2-hippocampus/sessions/` — historical record
- **CHANGELOG / RELEASE notes**: they document what happened when
- **Provider model caches**: `provider_models_cache.json`, `models_dev_cache.json` —
  auto-generated by Hermes, list all available models including old ones
- **Third-party code**: `.venv/`, `node_modules/` — not under your control
- **Benchmark configs**: `environments/benchmarks/` — pinned for reproducibility

### 3. Fix, don't preserve selectively

When a reference is found in an active file:

- **Config/routing value**: replace with the new model
- **Comparison text** ("old model X was slower"): remove the old model name,
  or rephrase without naming the specific old model
- **Model list in reference doc**: remove entries for models that are no longer
  the current routing target (they still exist on the provider, but listing
  them creates confusion about what's actively used)
- **Pedagogical example in a migration skill**: convert to a commented-out
  example with a "historical" label, or replace with a generic placeholder
  like `old-model-name`

Do NOT leave a reference saying "I'll keep this one because it's intentional."
If it's in an active file and an agent might read it, fix it. The only exception
is commented-out code or explicitly marked `# historical` lines.

### 4. Verify with final sweep

After all fixes, re-run the grep with the same patterns. Every remaining hit
should be either:
- Commented out with `#` or marked `historical`
- In an excluded file category (session archives, caches, CHANGELOG)
- A legitimate reference to a model's existence on the provider (only in a
  provider reference doc that explicitly lists what's available)

## Origin

Trigger: 2026-06-14 model routing change from multi-model (MiniMax + OpenRouter)
to single-model (opencode-go/deepseek-v4-flash). Initial pass left 4 "intentional"
references that the user corrected: "의도적 보류? 이것도 fix 해야 할 것 같은데."
Lesson: completeness over selective preservation.
