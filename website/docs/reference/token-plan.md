# MiniMax Token Plan

> **상태**: 활성 (2026-06-01 ~)
> **기본 모델**: `MiniMax-M3` (이전: `MiniMax-M2.7`)
> **마이그레이션**: 자동 (별도 작업 불필요)

Drewgent의 기본 LLM provider가 MiniMax M2.7에서 **M3**로 전환되었으며, 동시 **Token Plan** 신규 가격 정책이 적용됩니다. 이 문서는 변경 사항과 운영 가이드를 정리합니다.

---

## 1. 모델 변경 요약

| 항목 | 이전 (M2.7) | 이후 (M3) |
|------|-------------|-----------|
| 모델명 | `MiniMax-M2.7` | `MiniMax-M3` |
| 컨텍스트 윈도우 | 200K (204,800) | **1M (1,048,576)** — 5배 증가 |
| 컨텍스트 family default | 204,800 | 1,048,576 |
| 코딩 최적화 | 좋음 | 프론티어 코딩 (강화) |
| 멀티모달 | 텍스트 중심 | 텍스트 + 비전 + 도구 통합 |
| HuggingFace catalog | `MiniMaxAI/MiniMax-M2.5` | + `MiniMaxAI/MiniMax-M3` 추가 |

### Provider별 기본값

| Provider | Token Plan 적용 | 기본 모델 | 비고 |
|----------|----------------|----------|------|
| `minimax` (global) | ✅ Yes | `MiniMax-M3` | Token Plan quota |
| `minimax-cn` | ❌ No | `MiniMax-M3` | China endpoint, 별도 quota |
| 명시적 M2.5 | ❌ No | `MiniMax-M2.5` | 레거시 호환 |

## 2. 토큰 소모 방식 변경

> ⚠️ **상세 과금 정책 확인 필요** — M3 토큰 카운팅이 M2.7과 다를 수 있습니다. 운영 전 공식 문서로 확인 권장.

M3 + Token Plan 적용에 따라 토큰 소모 방식이 변경됩니다. 가능한 변경 축:

1. **카테고리 분리 청구**: input / output / cache_hit / reasoning / tool_use 가 분리 카운팅될 가능성
2. **하이브리드 과금**: 월정액 quota + 종량제 overage 동시 적용
3. **세션 ceiling**: per-conversation token 한도 (M3의 1M context 대비 quota 보호)

**확인 후 업데이트 필요 항목**:
- [ ] 정확한 단가 (1K tokens = $X?)
- [ ] Plan 종류별 quota (Free / Pro / Team / Enterprise)
- [ ] Cache hit discount 정책 (M2.7에 없던 기능일 수 있음)
- [ ] Tool use / function calling token 비용
- [ ] Reasoning / thinking tokens 별도 청구 여부
- [ ] 1M context 요청 시 surcharge 여부

운영 데이터가 쌓이면 본 섹션 업데이트 예정.

## 3. 1M Context Window 활용

M3의 1M context는 다음 작업에서 특히 유리합니다:

| 활용 | 효과 |
|------|------|
| 장기 세션 메모리 | P2-hippocampus 전체를 컨텍스트에 직접 로드 가능 |
| 코드베이스 분석 | vault 전체를 한 번에 ingest |
| 다중 보고서 비교 | 여러 문서를 동시에 cross-reference |
| RL training rollouts | `rl-training.md` 예시 (3 models × 16 completions × 3 steps) — context 여유 확보 |

**주의**:
- 1M 토큰 요청은 API latency 증가
- Token Plan quota 소모가 빠를 수 있음 (over-quota 시 overage 발생)
- 50K 이하 작업에 M2.5 사용 시 비용 효율적 (명시적 지정)

## 7. 트러블슈팅

### 모델 인식 실패

```text
Error: unknown model "MiniMax-M3"
```

→ `P0-brainstem/agent/model_metadata.py`의 `DEFAULT_CONTEXT_LENGTHS`가 업데이트됐는지 확인 (이번 패치로 자동 적용).

### Quota 초과

```text
Error: 429 Too Many Requests
```

→ Token Plan quota 소진. 콘솔에서 quota 확인 또는 M2.5로 fallback.

### M2.7 호환성 깨짐

기존 `MiniMax-M2.7`로 명시 호출 시 provider는 그대로 받지만, response 형식/성능이 M3와 다를 수 있음. M3로 마이그레이션 권장.
